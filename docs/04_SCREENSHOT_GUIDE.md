# Screenshot Placement Guide

Use clear browser screenshots with the address bar hidden if your college allows cropping. Keep all screenshots readable and consistently sized.

| Screenshot | Where to Capture | Use In Report | Use In PPT |
|---|---|---|---|
| Login Page | `/login` before signing in | Implementation chapter | Slide 12 |
| Registration Page | `/register` | User functions section | Slide 12 |
| User Dashboard | Login as a normal registered user | Results section | Slide 12 |
| Admin Dashboard | Login as `admin@example.com` | Admin module section | Slide 12 |
| User List | Admin dashboard user list area | Admin functions section | Slide 8 or 12 |
| Security Alerts | Trigger failed attempts, then view admin page | Security alert module | Slide 12 |
| Alert Resolution | Click alert toggle and capture resolved state | Testing/results section | Slide 12 |
| IDS Prediction Normal | `/ids/predict` with low-risk values | ML module section | Slide 10 or 12 |
| IDS Prediction Attack | `/ids/predict` with high-risk values | ML module section | Slide 10 or 12 |
| CSV Export | Click export link or show downloaded CSV | Reporting module section | Slide 13 |
| Database Tables | Open SQLite database and show table list | Database design section | Slide 8 |
| Test Output | Terminal showing `Ran 8 tests OK` | Testing chapter | Slide 13 |
| Model Metrics | Terminal/model metrics output | Results chapter | Slide 13 |
